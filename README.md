# otrack-pro (Omarchy Tracker Pro)

A high-performance, keyboard-centric screen time and productivity suite for Hyprland on Arch-based systems like Omarchy.

## 🚀 Features
- **Daemon (`otrackd`)**: A lightweight background service that tracks active window dwell time using Hyprland's IPC socket.
- **Deep Work Engine**: Actively blocks distracting apps during focus sessions.
- **Grace Period**: Configurable delay (e.g., 30s) before logging an app to prevent "Alt-Tab clutter."
- **Idle Detection**: Automatically pauses tracking when no user activity is detected.
- **CLI (`otrack`)**: A minimalist tool for status checks, reporting, and focus management.
- **TUI Dashboard**: A full-screen terminal dashboard built with `ratatui`.
- **Waybar Integration**: JSON output support for custom status modules.
- **Privacy First**: All your data is stored locally in a SQLite database.

## 🛠️ Installation

### Prerequisites
- **Rust & Cargo** (Standard build tools)
- **Hyprland** (Required for window tracking)
- **SQLite** (Runtime database)

### 1. Build from Source
```bash
cargo build --release
```

### 2. Install Binaries
To make the commands available globally:
```bash
sudo cp target/release/otrack target/release/otrackd /usr/local/bin/
```

### 3. Systemd Service (Recommended)
Ensure the daemon starts automatically on login:
```bash
mkdir -p ~/.config/systemd/user/
cp otrackd.service ~/.config/systemd/user/
systemctl --user daemon-reload
systemctl --user enable --now otrackd
```

## 💻 Usage

### CLI Commands
- `otrack status`: View the active app, current session duration, and focus mode status.
- `otrack report`: Display a formatted table of today's application usage.
- `otrack start <minutes>`: Begin a focus session for a specified duration.
- `otrack stop`: Manually end an active focus session.
- `otrack dashboard`: Open the interactive terminal UI.
- `otrack waybar`: Output JSON for a custom Waybar module.

### Focus Mode
During a focus session, any app in your `blacklist` will be automatically closed if it gains focus, helping you stay in the flow.

## ⚙️ Configuration

The configuration file is located at `~/.config/otrack/config.toml`. It is created automatically with defaults on the first run.

### Sample Config
```toml
[general]
database_path = "~/.local/share/otrack/usage.db"
idle_timeout = 300   # Seconds of inactivity before pausing (Default: 5m)
grace_period = 30   # Seconds before a window is logged (Default: 30s)

[blacklist]
apps = ["discord", "slack", "spotify"]
block_during_focus = true

[work_apps]
apps = ["foot", "neovim", "zed"]
```

### ⚠️ Applying Changes
The configuration is loaded into memory when the daemon starts. **You must restart the daemon for any changes to take effect:**
```bash
systemctl --user restart otrackd
```

## 🧿 Eye-health and Repetitive Strain Injury (RSI) Safety

Otrack includes optional screen-time reminders to help reduce eye strain and repetitive stress injuries caused by prolonged uninterrupted computer use.

When enabled, Otrack will send a desktop notification after every **N minutes of continuous screen activity** (defined as time spent not idle). This can help encourage regular micro-breaks and reduce fatigue.

### Enabling Screen-Time Alerts

To enable reminders, add the following parameter to your `config.toml`:

```toml
[general]
work_alert_minutes = 25
```

This will notify you every 25 minutes of continuous activity.

> ⚠️ After modifying your configuration, you must restart the daemon for changes to take effect:

```bash
systemctl --user restart otrackd
```

If you do not restart the daemon, Otrack will continue using the previously loaded configuration.

---

### How It Works

* Screen-time is measured based on Otrack’s existing idle detection.
* Time resets when the user becomes idle (no activity for `idle_timeout` seconds).
* Notifications are opt-in — if the parameter is not present, no reminders are sent.

## 💾 Persistence & Data

- **Local Storage**: All data is saved to a SQLite database at `~/.local/share/otrack/usage.db`.
- **Session Survival**: Your history is preserved across reboots and shutdowns.
- **Graceful Shutdown**: The daemon captures system termination signals to ensure the current session is saved before the process exits.
- **Daily Refresh**: Reports generated by `otrack report` automatically filter for the current day, though historical data is always retained in the database.

## 🛡️ Privacy
**otrack-pro** does not connect to the internet. Your usage data never leaves your machine.
